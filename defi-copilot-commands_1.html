Technical Q&A
How does it actually work?
Your 4 most important questions answered honestly.
1
Is it Claude API or code that performs the trades?
+
Both — they each do a different job.

Claude API is the brain. It reads what the user typed and returns structured instructions like { action: "swap", tokenIn: "ETH", amount: 500 }. Claude never touches money or executes anything.

Your backend code is the hands. It receives Claude's output, calls the right trading API (1inch, Jupiter, Gelato), builds the transaction calldata, and sends it to the user's wallet for signing.

The user's wallet is the lock. Nothing executes until the user approves in MetaMask. Private keys never leave their device.
User types: "Swap $500 ETH to USDC"
↓
Claude API → parses intent → returns JSON params
↓
Your backend → calls 1inch → builds calldata
↓
User wallet → user signs in MetaMask → tx broadcasts
↓
Blockchain → atomic swap executes → tokens arrive
2
Do I need to pay for all the third-party service APIs?
+
Most are completely free to start. You only pay as you scale.

Free to integrate: 1inch API, Jupiter, LI.FI bridge SDK, Alchemy (300M units/month free), Helius Solana, CoinGecko, Railgun, Umbra Protocol, DeFiLlama — all completely free at MVP scale.

Total burn before revenue: ~$80–120/month. You can validate the entire product for the cost of a dinner out.
3
What happens to limit orders when the user closes the browser?
+
Your server keeps running 24/7 — the user's browser is completely irrelevant.

When a user sets a limit order, they sign it once (off-chain, zero gas). The signed bytes are stored in your database on Railway. A BullMQ background job on your server checks Chainlink price feeds every 12 seconds — around the clock, whether the user is online or not.

When the trigger fires, your server broadcasts the pre-signed transaction. Swap executes. User wakes up with a push notification: "Your SOL order filled at $99.80."

For DCA and recurring automation, Gelato Network acts as the decentralized executor — it monitors and triggers even if your server has brief downtime.
User sets order → signs once → closes browser
↓
YOUR SERVER (Railway) → runs 24/7 without stopping
↓ checks every 12 seconds via Chainlink
SOL price hits $99.80 → condition triggered ✓
↓
Server broadcasts pre-signed tx to blockchain
↓
User notified → "SOL order filled at $99.80 ✓"
4
Which AI model should power the copilot?
+
Use different models for different jobs — not one model for everything.

Claude Sonnet → primary brain for intent parsing. Best at following complex instructions and returning reliable JSON. "Buy $200 ETH only if gas is under 15 gwei" — Claude handles multi-condition logic without errors.

GPT-4o audio → voice commands. Its audio transcription is the best available. Users can speak trades on mobile — nobody else in DeFi has this feature.

Perplexity API → live market context. When users ask "what's happening with SOL today?" — Perplexity searches current news. Claude alone has no live internet access.

Budget: ~$50–80/month at 1,000 active users. AI is your cheapest cost and your biggest differentiator.
